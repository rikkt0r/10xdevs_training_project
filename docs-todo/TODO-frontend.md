  ## Trade-offs & Considerations                                                                                                                                                                                                                 
                                                                                                                                                                                                                                                 
  ### RTK Query Migration                                                                                                                                                                                                                        
  **Decision:** Keep current service layer pattern (plain Axios)                                                                                                                                                                                 
  **Rationale:**                                                                                                                                                                                                                                 
  - Current pattern is working well and consistent                                                                                                                                                                                               
  - RTK Query migration would be a large refactor                                                                                                                                                                                                
  - Can migrate incrementally later if needed                                                                                                                                                                                                    
  - Focus on feature completion first                                                                                                                                                                                                            
                                                                                                                                                                                                                                                 
  ### React Hook Form                                                                                                                                                                                                                            
  **Decision:** Keep current validation pattern initially                                                                                                                                                                                        
  **Rationale:**                                                                                                                                                                                                                                 
  - Current pattern is simple and works                                                                                                                                                                                                          
  - Can introduce React Hook Form for complex forms (board/inbox config)                                                                                                                                                                         
  - Avoid over-engineering for simple forms                                                                                                                                                                                                      
                                                                                                                                                                                                                                                 
  ### Component Library                                                                                                                                                                                                                          
  **Decision:** Build custom components on top of React Bootstrap                                                                                                                                                                                
  **Rationale:**                                                                                                                                                                                                                                 
  - Consistency with existing pages                                                                                                                                                                                                              
  - Full control over styling and behavior                                                                                                                                                                                                       
  - Easier to maintain than mixing libraries                                                                                                                                                                                                     
  - Can extract to separate package later if needed                                                                                                                                                                                              
                                                                                                                                                                                                                                                 
  ### State Management                                                                                                                                                                                                                           
  **Decision:** Redux for global state, local state for page-specific data                                                                                                                                                                       
  **Rationale:**                                                                                                                                                                                                                                 
  - Follows existing pattern                                                                                                                                                                                                                     
  - Keeps components simple                                                                                                                                                                                                                      
  - Avoids over-centralization                                                                                                                                                                                                                   
                                                                                                                                                                                                                                                 
  ---                                                                                                                                                                                                                                            
                                                                                                                                                                                                                                                 
  ## Risks & Mitigation                                                                                                                                                                                                                          
                                                                                                                                                                                                                                                 
  **Risk:** Large number of files to create (75+)                                                                                                                                                                                                
  **Mitigation:** Phase approach, start with foundation components, reuse patterns                                                                                                                                                               
                                                                                                                                                                                                                                                 
  **Risk:** Breaking existing pages during refactor                                                                                                                                                                                              
  **Mitigation:** Refactor one page at a time, test thoroughly                                                                                                                                                                                   
                                                                                                                                                                                                                                                 
  **Risk:** Inconsistent styling across new components                                                                                                                                                                                           
  **Mitigation:** Create design system document, use Bootstrap variables                                                                                                                                                                         
                                                                                                                                                                                                                                                 
  **Risk:** Performance issues with large ticket lists                                                                                                                                                                                           
  **Mitigation:** Implement pagination, virtual scrolling if needed
